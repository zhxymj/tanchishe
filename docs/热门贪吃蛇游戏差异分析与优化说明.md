# 热门贪吃蛇游戏差异分析与本项目优化说明

## 参考对象

1. Snake.io：偏移动端休闲竞技，强调简单上手、明亮视觉、排行榜、离线游玩、皮肤和短局反馈。
2. slither.io：偏多人在线竞技，强调同场对抗、发光食物、体型成长、速度变化和即时视觉反馈。

## 主要差异

| 对比项 | Snake.io | slither.io | 本项目取舍 |
| --- | --- | --- | --- |
| 核心体验 | 单局节奏快，规则清晰 | 多人同场竞争，压迫感强 | 保留单机标准贪吃蛇规则，不做联网对抗 |
| 操作方式 | 触屏摇杆或滑动控制 | 鼠标/键盘控制移动方向 | 使用方向键和 `W/A/S/D`，去除移动端轮盘 |
| 视觉表现 | 色彩明亮，反馈直接 | 深色背景、发光食物、蛇身成长明显 | 使用 raylib 窗口、深色街机背景、网格、发光食物和圆角蛇身 |
| 成长反馈 | 吃食物加分、变长、冲榜 | 吃光点变长，长度是主要压力来源 | 实现吃食物增长、分数、最高分、等级和速度递增 |
| 商业内容 | 皮肤、活动、广告 | 排名、皮肤、多人房间 | 不做商业系统，聚焦课程可实现的图形化单机小游戏 |

## 本项目已做优化

1. 图形化窗口：使用 `raylib` 实现 `960x720` 窗口，不再是控制台字符版。
2. 现代街机界面：深色背景、轻微网格纹理、棋盘阴影、边框和右侧信息面板。
3. 蛇身表现：圆角蛇身节点、颜色层次、蛇头眼睛和方向感。
4. 食物反馈：食物使用发光和脉冲动画，吃到食物后有短暂视觉反馈和音效。
5. 状态完整：包含主菜单、运行中、暂停、游戏结束四种状态。
6. 交互完整：支持方向键、`W/A/S/D`、`Space/P` 暂停继续、`R` 重开、`M` 返回菜单、`Tab` 切换难度，以及鼠标点击按钮。
7. 分数系统：显示当前分数、最高分、等级和速度条。
8. 难度变化：支持 Easy、Normal、Hard 三档难度，分数越高蛇移动间隔越短。
9. 音效和数据：支持音效开关、最高分重置，并使用 `snake_raylib_highscore.dat` 保存最高分。
10. 结构优化：拆分为 `main.c`、`game.c`、`snake.c`、`food.c`、`ui.c`、`controls.c` 等模块，方便五人分工。

## 未采用的功能

1. 多人联网：会显著增加网络同步、房间和碰撞规则复杂度，不适合当前课程项目。
2. 移动端轮盘：用户要求转换成电脑控制，因此去除轮盘。
3. 商店、广告、活动：不符合课程实践目标。
4. 复杂皮肤系统：可作为后续扩展，但当前版本优先保证核心玩法和窗口化 UI。
