# 热门贪吃蛇游戏差异分析与本项目优化说明

## 参考对象

1. Snake.io：偏移动端休闲竞技，强调简单上手、排行榜、离线游玩、皮肤和活动内容。
2. slither.io：偏多人在线竞技，强调与其他玩家同场竞争、吞食发光球、加速冲刺和体型成长。

## 主要差异

| 对比项 | Snake.io | slither.io | 对本项目的启发 |
| --- | --- | --- | --- |
| 核心体验 | 轻量、节奏快、移动端友好 | 在线竞技、对抗强、成长反馈明显 | 控制台版应保持操作简单，同时增加分数和成长反馈 |
| 食物设计 | 吃食物变长、冲榜 | 吃发光球变长，也可从其他玩家处获得收益 | 增加奖励食物，让得分变化更明显 |
| 速度机制 | 难度随分数提升 | 可主动加速，但通常有代价 | 增加主动冲刺，用蛇身长度换速度 |
| 长期目标 | 排行榜、活动、皮肤 | 排名、体型成长、多人压迫感 | 增加历史最高分，形成重复游玩目标 |
| 适配性 | 适合手机短局 | 适合联网多人对抗 | 课程 C 项目不做联网，保留可实现的单机优化 |

## 本项目已做优化

1. 历史最高分：新增 `snake_highscore.dat` 保存最高分，重新运行游戏后仍能显示。
2. 奖励食物：普通食物显示为 `*`，奖励食物显示为 `$`，奖励食物分数更高。
3. 加速冲刺：按 `B` 或空格可以短时间加速，但会消耗 1 节蛇身长度，参考多人贪吃蛇中的“加速有代价”机制。
4. 界面反馈：顶部显示当前分数、最高分、等级、速度、蛇身长度和冲刺剩余时间。
5. 输入优化：支持 `W/A/S/D` 和方向键两种移动方式。

## 未采用的功能

1. 多人联网：超出普通 C 控制台课程项目范围，实现成本高。
2. 皮肤系统：对算法和课程实践帮助较小。
3. 商城、活动、广告等商业功能：不适合课程项目。
